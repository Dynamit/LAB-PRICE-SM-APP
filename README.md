# מחשבון מחירי מעבדה — אפליקציית מובייל (PWA)

אפליקציה לטלפונים (iPhone + Android) לחישוב הצעות מחיר לבדיקות מעבדה, עם ייצוא
הצעת מחיר ללקוח ורשימת בדיקות לצוות. **עובדת לגמרי ללא אינטרנט** אחרי ההתקנה.

זוהי גרסת PWA של מחשבון המעבדה — **קוד אחד** שרץ על שתי הפלטפורמות, ללא חנויות
אפליקציות וללא צורך ב-Mac. הנתונים והממשק מקורם בפרויקט המקורי (lab-price-calc);
כאן הם עטופים כ-PWA עצמאי ומותקן.

## איך זה עובד (קצר)

- אתר סטטי (HTML/CSS/JS) **ללא build**, מתארח על GitHub Pages (HTTPS).
- `service-worker.js` שומר מראש (precache) את כל הקבצים — קוד, נתונים, גופנים,
  אייקונים וספריות ה-PDF — כך שאחרי טעינה ראשונה אחת האפליקציה רצה offline מלא.
- הגופנים (Heebo, Frank Ruhl Libre) מוטמעים מקומית ב-`assets/fonts/` (אין תלות
  ב-Google Fonts).
- ייצוא PDF נעשה במכשיר עצמו (`html2canvas` + `jsPDF`, מוטמעים ב-`assets/lib/`)
  ומשותף דרך תפריט השיתוף של המכשיר (וואטסאפ/מייל) או נפתח לשמירה.

## הרצה מקומית (לפיתוח)

ה-`fetch` של קובצי ה-JSON וה-Service Worker דורשים שרת (לא לפתוח `index.html`
ישירות כ-`file://`):

```bash
python -m http.server 8000
```
ואז לפתוח http://localhost:8000 (רענון אחרי שינוי: `Ctrl+F5`).

## פרסום (Deploy)

ה-repo כבר באוויר על GitHub Pages:

### 👉 https://dynamit.github.io/LAB-PRICE-SM-APP/

כל `git push` ל-`main` מעדכן את האתר אוטומטית (תוך כדקה). ראו «עדכון נתונים / קוד
בעתיד» למטה.

## התקנה על הטלפון

מדריך פשוט לצוות (אחיות) נמצא בקובץ **[הוראות-התקנה.md](הוראות-התקנה.md)** — כדאי
להעביר אותו / לצרף לינק ל-WhatsApp. בקצרה:

פותחים את הכתובת פעם אחת **עם אינטרנט**, ואז:

- **iPhone (חובה Safari):** כפתור «שיתוף» ⬆️ → «הוסף למסך הבית».
- **Android (Chrome):** הבאנר «התקן» שמופיע למטה, או תפריט ⋮ → «התקנת אפליקציה».

האייקון מופיע על מסך הבית והאפליקציה נפתחת במסך מלא. מכאן והלאה — עובדת גם
ללא רשת.

## עדכון נתונים / קוד בעתיד

1. לעדכן את קובצי ה-JSON ב-`assets/data/` (ידנית, או דרך `tools/build_data.py`
   מקובץ Excel — ראו למטה).
2. **להעלות את `CACHE_VERSION` ב-`service-worker.js`** (למשל `"v1"` → `"v2"`).
   זה הצעד שמפעיל את העדכון אצל המשתמשים.
3. `git push`. GitHub Pages מתעדכן תוך כדקה.
4. בפעם הבאה שהמשתמש פותח את האפליקציה **עם רשת**, יופיע באנר «יש גרסה חדשה» →
   לחיצה על «טען מחדש» מביאה את הנתונים החדשים. בלי חנות, בלי אישורים.

### עדכון מה-Excel (אופציונלי)

קובצי ה-Excel אינם נשמרים ב-repo (מוחרגים ב-`.gitignore`) — רק ה-JSON שנוצר.

```bash
pip install openpyxl      # פעם אחת
python tools/build_data.py
```
מיפוי העמודות מתועד בתוך `tools/build_data.py`.

## מבנה

| נתיב | תפקיד |
|------|-------|
| `index.html` / `style.css` / `script.js` | האפליקציה |
| `manifest.webmanifest` | הגדרות PWA (שם, אייקונים, מצב standalone) |
| `service-worker.js` | מטמון offline + מנגנון עדכון (`CACHE_VERSION`) |
| `assets/data/*.json` | מחירון + ספרי בדיקות (רמת החייל / חיפה) |
| `assets/fonts/` | גופנים מקומיים (woff2) |
| `assets/icons/` | אייקוני אפליקציה |
| `assets/lib/` | jsPDF + html2canvas (לייצוא PDF offline) |
| `tools/build_data.py` | ממיר Excel → JSON לעדכונים |

## הערות

- iOS לפעמים מעכב הפעלת Service Worker חדש — באנר «יש גרסה חדשה» + «טען מחדש»
  פותר זאת.
- ה-PDF נוצר כתמונה (כדי שעברית/RTL יֵצאו מושלמים), ולכן הטקסט בו אינו ניתן
  לסימון — מתאים לכלי פנימי.
